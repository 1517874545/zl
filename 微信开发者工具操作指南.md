# 微信开发者工具操作指南

## 一、打开微信开发者工具

### 步骤1：启动工具
1. 在电脑上找到「微信开发者工具」图标
2. 双击打开（如果没有安装，需要先下载安装）

### 步骤2：登录账号
1. 打开后，使用微信扫码登录
2. 或者使用账号密码登录

## 二、配置本地设置（跳过域名校验）

### 步骤1：进入项目详情
1. 在微信开发者工具中，点击右上角的「详情」按钮
   - 位置：工具栏最右侧，通常显示为「详情」或齿轮图标

### 步骤2：打开本地设置
1. 在弹出的详情窗口中，点击「本地设置」标签页
   - 标签页通常在窗口顶部，包括：项目配置、本地设置等

### 步骤3：勾选跳过校验
1. 在「本地设置」页面中，找到「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」选项
2. 勾选这个复选框（打勾 ✓）
3. 关闭详情窗口

**说明：**
- 这个设置只在开发阶段有效，不会影响正式发布
- 勾选后可以跳过域名校验，方便本地开发测试

## 三、导入项目

### 方法一：新建项目（推荐）

#### 步骤1：点击新建
1. 在微信开发者工具主界面，点击「+」按钮或「新建项目」按钮
   - 通常在左上角或中央位置

#### 步骤2：填写项目信息
在弹出的新建项目窗口中，填写以下信息：

1. **项目名称**：`校园生活管理` 或 `campus-life`
   - 可以自定义，不影响功能

2. **目录**：点击「选择」或「浏览」按钮
   - 选择项目文件夹：`D:\aer\wenjian`
   - 或者直接输入路径：`D:\aer\wenjian`

3. **AppID**：
   - 如果有正式 AppID，选择「使用正式 AppID」
   - 如果没有，选择「使用测试号」或「不使用云服务」
   - 测试号可以用于开发测试

4. **开发模式**：选择「小程序」

#### 步骤3：确认创建
1. 检查所有信息是否正确
2. 点击「新建」或「确定」按钮
3. 等待项目加载完成

### 方法二：导入已有项目

#### 步骤1：打开项目
1. 在微信开发者工具主界面，点击「导入项目」或「打开项目」
2. 选择项目目录：`D:\aer\wenjian`
3. 填写 AppID（同上）
4. 点击「导入」或「确定」

## 四、编译项目

### 步骤1：检查项目结构
导入后，左侧文件树应该显示：
```
miniprogram/
  ├── app.ts
  ├── app.json
  ├── pages/
  ├── utils/
  └── ...
```

### 步骤2：点击编译
1. 在工具栏找到「编译」按钮
   - 位置：顶部工具栏，通常是最左侧的绿色按钮
   - 图标可能显示为「▶」或「编译」

2. 点击「编译」按钮
   - 或者使用快捷键：`Ctrl + B`（Windows）或 `Cmd + B`（Mac）

### 步骤3：查看编译结果
1. 等待编译完成（通常几秒钟）
2. 如果编译成功：
   - 右侧预览窗口会显示小程序界面
   - 底部 Console 面板显示「编译成功」或类似信息
3. 如果编译失败：
   - 查看 Console 面板的错误信息
   - 根据错误提示修复问题

## 五、常见问题

### 问题1：找不到「详情」按钮
**解决方法：**
- 确保项目已打开
- 查看工具栏最右侧
- 或者点击菜单栏「项目」→「项目设置」

### 问题2：找不到「本地设置」
**解决方法：**
- 在详情窗口中，查看顶部标签页
- 可能显示为「本地设置」或「Local Settings」
- 如果没有，可能是版本问题，尝试更新微信开发者工具

### 问题3：项目导入失败
**解决方法：**
1. 检查项目目录是否正确
2. 确认 `miniprogram` 文件夹存在
3. 确认 `app.json` 文件存在
4. 检查是否有权限访问该目录

### 问题4：编译报错
**常见错误及解决方法：**

**错误1：找不到 app.json**
- 检查项目目录结构
- 确认 `miniprogram/app.json` 文件存在

**错误2：TypeScript 编译错误**
- 检查 `tsconfig.json` 配置
- 确认所有类型定义正确

**错误3：模块找不到**
- 检查导入路径是否正确
- 确认文件是否存在

## 七、界面说明

### 微信开发者工具界面布局

```
┌─────────────────────────────────────────┐
│  工具栏  [编译] [预览] [上传] [详情]    │
├──────────┬──────────────────────────────┤
│          │                               │
│  文件树  │      预览窗口                 │
│          │      (小程序界面)             │
│          │                               │
│          │                               │
├──────────┴──────────────────────────────┤
│  Console | Network | Storage | ...      │
│  (调试面板)                              │
└─────────────────────────────────────────┘
```

### 关键按钮位置

- **编译按钮**：顶部工具栏最左侧，绿色播放图标
- **详情按钮**：顶部工具栏最右侧，齿轮或设置图标
- **预览窗口**：右侧大窗口，显示小程序运行效果
- **Console 面板**：底部面板，显示日志和错误信息

## 八、验证设置是否生效

### 测试方法：
1. 编译项目后，打开小程序
2. 打开 Console 面板（底部）
3. 尝试操作需要网络请求的功能（如加载动态）
4. 查看 Console 是否有网络错误
5. 如果之前有域名错误，现在应该消失了

## 九、下一步

设置完成后，可以：
1. 开始测试各个功能模块
2. 查看 Console 日志，排查问题
3. 使用 Network 面板查看网络请求
4. 参考「测试指南.md」进行功能测试
